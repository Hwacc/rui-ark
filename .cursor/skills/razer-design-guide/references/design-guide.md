# Razer 通用设计指南
本文档定义 Razer 主题下的通用设计原则与规范，适用于 .pen 文件设计及 UI 组件实现。具体颜色、尺寸、组件状态等请参阅对应 reference 文档。

## 设计原则

### 1. 品牌一致性
- 所有设计元素必须符合 Razer 品牌调性：科技感、电竞风格、深色主题
- 统一使用定义的颜色值，禁止硬编码任意颜色
- 保持与 Razer 官方商店 (razer.com/store) 的视觉语言一致

### 2. 层次与对比
- **背景层次**：主背景 → 二级背景 → 提升表面，形成清晰层级
- **文字层次**：主文字与次要文字应区分信息优先级
- **对比度**：深色背景上使用白色/浅色文字；浅色或 primary 背景上使用黑色文字，确保可读性

### 3. 反馈与状态
- 每个可交互元素需具备完整状态：默认、悬停、激活/选中、焦点、禁用
- 状态变化应清晰可辨，使用颜色、边框、透明度等传达
- 焦点状态必须可见，满足可访问性要求

### 4. 尺寸与比例
- 遵循尺寸档位系统 (xs、sm、base、lg)，保持档位间比例关系
- 同一组件不同档位间保持视觉连续性
- 尺寸与档位对应，避免随意取值
- 留白与间距合理，避免拥挤

### 5. 克制与专注
- 主强调色用于关键操作与状态，避免过度使用
- 次要元素使用 secondary 色系，不喧宾夺主

## 通用规范

### 可访问性
- 文字与背景对比度满足 WCAG 最低要求
- 焦点指示器清晰可见 (2px outline)
- 禁用状态通过透明度与颜色区分，不依赖单一视觉线索

### 布局
- 使用尺寸档位对应的间距 (xs/sm/base/lg) 保持节奏一致
- 对齐方式统一（左对齐、居中对齐等根据场景选择）

### 阴影
- 深色背景下阴影适度，避免过重
